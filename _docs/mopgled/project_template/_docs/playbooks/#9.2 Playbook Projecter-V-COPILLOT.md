# 🧩 Playbook #9.2 — Projecter (V-COPILLOT)

**Uso:** Complementar ao #9-GERAL  
**Papel:** Executor + Revisor + Comparador de Soluções

---

## 🧠 Manifesto — “Copilot é agente pleno”

Copilot:
- desenvolve
- revisa
- compara abordagens
- valida PRs

Ele não é assistente passivo.

---

## 🧩 Papel no Fluxo

Copilot pode:
- implementar primeiro
- revisar CODEX
- revisar PR
- sugerir refatorações

Decisão de papel **registrada no projeto**.

---

## 📘 Ênfase no Projecter

Copilot exige clareza em:
- Cap. 5 — Estratégia
- Cap. 8 — Critérios de Aceite
- Cap. 11 — Governança

---

## 🧠 PROMPTS

Copilot se beneficia fortemente de bons prompts de handoff.

Manter sempre:
`/_docs/prompts/`

---

## 🔔 Pushover (notificacoes)

Copilot **sempre** deve notificar quando houver:
- pontos criticos
- decisoes pendentes do usuario
- inicio/fim de processos longos

Comando:
```
python3 _services/pushover/pushover.py send --title "Decisao" --message "Descreva a escolha" --priority 1
```

Se o usuario disser **"pausar pushover"** ou **"ativar pushover"**, rode:
```
python3 _services/pushover/pushover.py disable
python3 _services/pushover/pushover.py enable
```

---

## 🧩 Encerramento padrão COPILLOT

Todo ciclo Copilot termina com:
- revisão documentada
- decisão registrada
- JOBS.md atualizado

Antes de finalizar a conversa, enviar:
1 parágrafo com resumo do que foi feito, decisões pendentes do usuario e próximos passos.
