# 🧩 Playbook #9.1 — Projecter (V-CODEX)

**Uso:** Complementar ao #9-GERAL  
**Papel:** Executor intensivo / Revisor secundário

---

## 🧠 Mentalidade CODEX

CODEX executa rápido e em volume.  
Por isso, **depende de escopo claro**.

Antes de codar, CODEX deve saber:
- quais arquivos mexer
- o que não mexer
- qual é o critério de aceite

Sem isso → parar.

---

## 📘 Ênfase no Projecter

CODEX exige atenção máxima em:
- Cap. 4 — Escopo
- Cap. 6 — Arquitetura
- Cap. 7 — Fluxos
- Cap. 8 — Aceite

---

## 🧠 JOBS.md

CODEX **sempre**:
- cria tarefa ao iniciar
- fecha tarefa ao concluir

Sem atualização do JOBS.md → trabalho incompleto.

---

## 🔔 Pushover (notificacoes)

CODEX **sempre** deve notificar quando houver:
- pontos criticos
- decisoes pendentes do usuario
- inicio/fim de processos longos

Comando:
```
python3 _services/pushover/pushover.py send --title "Decisao" --message "Descreva a escolha" --priority 1
```

Se o usuario disser **"pausar pushover"** ou **"ativar pushover"**, rode:
```
python3 _services/pushover/pushover.py disable
python3 _services/pushover/pushover.py enable
```

---

## 🧩 Encerramento padrão CODEX

Todo trabalho CODEX termina com:
- código entregue
- JOBS.md atualizado
- aguardando revisão (Copilot ou humano)

Antes de finalizar a conversa, enviar:
1 parágrafo com resumo do que foi feito, decisões pendentes do usuario e próximos passos.
